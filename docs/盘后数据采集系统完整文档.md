## 系统概述

### 目标

构建一个**全自动、高可靠、可追溯**的盘后数据采集系统，实现：

1. **每日 16:00 自动更新** 1383 只股票的盘后数据
2. **双 API 备份**（新浪 + 东方财富）确保 100% 成功率
3. **自动追加到 CSV**，避免重复数据
4. **自动发送飞书通知**，实时掌握执行状态
5. **完整日志记录**，支持问题追溯

### 核心价值

- **自动化**：无需人工干预，每日自动执行
- **标准化**：统一的数据格式和筛选标准
- **可视化**：交互式仪表盘，支持 K 线图分析
- **可复制**：完整的文档和脚本，确保每次筛选逻辑一致

### 数据流程

```
原始数据源 (Tushare/新浪财经)
    ↓
数据更新脚本 (update-daily-data.js)
    ↓
CSV 历史数据 (data/merged/)
    ↓
股票筛选脚本 (screen-stocks.js)
    ↓
Excel 筛选结果 (output/excel/)
    ↓
仪表盘生成 (generate-dashboard.js)
    ↓
HTML 仪表盘 (output/dashboard/)
    ↓
飞书通知 (send-notification.js)
```

## 数据更新流程

### 执行时间

**每日 16:00**（A 股收盘后 1 小时）

### 详细步骤

#### 步骤 1: 读取股票列表

```javascript
// 从 stock-pool-from-total-stocks.json 读取 1383 只股票
const stockPool = JSON.parse(fs.readFileSync('stock-pool-from-total-stocks.json'));
console.log(`📊 股票数：${stockPool.length}只`);
```

**股票池构成**:
- 上海主板：792 只
- 深圳主板：591 只
- **总计**: 1383 只

#### 步骤 2: 分批获取数据

```javascript
// 每批 20 只，间隔 3 秒
const batchSize = 20;
for (let i = 0; i < stocks.length; i += batchSize) {
    const batch = stocks.slice(i, i + batchSize);
    await processBatch(batch);
    await sleep(3000);  // 避免 API 限流
}
```

**API 选择**:
- **主 API**: 新浪财经（速度快，成功率高）
- **备用 API**: 东方财富（新浪失败时切换）

#### 步骤 3: 解析 API 响应

**新浪 API 字段说明**:

| 索引 | 字段 | 说明 | 示例 |
|------|------|------|------|
| `[0]` | 股票名 | 股票名称 | 工商银行 |
| `[1]` | 开盘价 | 今日开盘 | 6.90 |
| `[2]` | 昨收价 | 昨日收盘 | 6.92 |
| `[3]` | 当前价 | 最新价格 | 6.92 |
| `[4]` | 最高价 | 今日最高 | 6.95 |
| `[5]` | 最低价 | 今日最低 | 6.88 |
| `[8]` | 成交量 | 成交股数 | 802810 |
| `[9]` | 成交额 | 成交金额 | 781977 |
| `[30]` | 日期 | 交易日期 | 2026-03-02 |
| `[31]` | 时间 | 交易时间 | 15:00:03 |

**数据格式转换**:
```javascript
{
    ts_code: "601398.SH",
    trade_date: "20260302",
    open: 6.90,
    close: 6.92,
    high: 6.95,
    low: 6.88,
    vol: 8028,
    amount: 7819.77,
    pct_chg: 0.06
}
```

#### 步骤 4: 追加到 CSV

// 检查是否已存在该日期数据

## 定时任务配置

### Windows 任务计划程序

#### 创建工作流任务

```powershell
# 以管理员身份运行 PowerShell

$action = New-ScheduledTaskAction -Execute "node" `
    -Argument "scripts/workflow/run-full-workflow.js" `
    -WorkingDirectory "C:\Users\Bob\.openclaw\workspace\stock-data"

$trigger = New-ScheduledTaskTrigger -Weekly `
    -DaysOfWeek Monday,Tuesday,Wednesday,Thursday,Friday `
    -At 4pm

$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries `
    -DontStopIfGoingOnBatteries `
    -StartWhenAvailable `
    -ExecutionTimeLimit (New-TimeSpan -Hours 1)

Register-ScheduledTask `
    -TaskName "StockAnalysisWorkflow" `
    -TaskPath "\Stock\" `
    -Action $action `
    -Trigger $trigger `
    -Settings $settings `
    -RunLevel Highest `
    -Force
```

#### 验证任务

```powershell
# 查看任务状态
Get-ScheduledTask -TaskName "StockAnalysisWorkflow" -TaskPath "\Stock\"

# 查看任务历史
Get-ScheduledTaskInfo -TaskName "StockAnalysisWorkflow" -TaskPath "\Stock\"

# 手动触发测试
Start-ScheduledTask -TaskName "StockAnalysisWorkflow" -TaskPath "\Stock\"
```

## 故障排查

### 问题 1: 数据更新失败

```bash
# 1. 检查 API 可用性
node scripts/archive/test-api.js

# 2. 手动执行数据更新
node scripts/workflow/update-daily-data.js

# 3. 查看详细日志
Get-Content logs\update-*.log -Tail 100
```

### 问题 2: 股票筛选结果为空

```bash
# 1. 检查 CSV 数据是否最新
Get-Content data\merged\sh_main.csv -Head 5

# 2. 检查筛选脚本
node scripts/workflow/screen-stocks.js
```

### 问题 3: 飞书通知未发送

```bash
# 1. 测试飞书配置
node scripts/workflow/send-notification.js

# 2. 检查 App Secret 是否过期
# 3. 验证 Chat ID 是否正确
```
