## 🎯 目标

构建一个**全自动、高可靠、可追溯**的股票分析工作流，实现：

1. **每日 16:00 自动更新**盘后数据
2. **自动筛选**1383 只基础股票池中的高分股票
3. **自动生成**最新仪表盘
4. **自动通知**执行结果（飞书/微信）
5. **完整日志**记录，支持问题追溯

## 🔧 核心组件设计

### 1. 工作流编排脚本 (`run-full-workflow.js`)

**职责**: 顺序执行各阶段，处理错误，记录日志

```javascript
// 伪代码结构
const workflow = {
    stages: [
        {
            name: '数据更新',
            script: 'update-daily-data.js',
            timeout: 1800000,  // 30 分钟
            retry: 2,          // 失败重试 2 次
            critical: true     // 关键阶段，失败则终止
        },
        {
            name: '股票筛选',
            script: 'full-analysis-industry-top5.js',
            timeout: 60000,    // 1 分钟
            retry: 1,
            critical: true
        },
        {
            name: '仪表盘生成',
            script: 'generate-dashboard.js',
            timeout: 60000,
            retry: 1,
            critical: false    // 非关键，失败可继续
        },
        {
            name: '结果通知',
            script: 'send-feishu-notification.js',
            timeout: 10000,
            retry: 3,
            critical: false
        }
    ],
    
    async run() {
        const startTime = Date.now();
        const results = [];
        
        for (const stage of this.stages) {
            const result = await this.executeStage(stage);
            results.push(result);
            
            if (!result.success && stage.critical) {
                console.error(`❌ 关键阶段失败：${stage.name}`);
                break;
            }
        }
        
        await this.generateReport(results, startTime);
    }
};
```

### 4. 错误处理机制

#### 4.1 重试策略

```javascript
async function executeWithRetry(script, config) {
    let lastError;
    
    for (let attempt = 1; attempt <= config.retry + 1; attempt++) {
        try {
            console.log(`▶️ 执行 ${script} (尝试 ${attempt}/${config.retry + 1})`);
            await executeScript(script, config.timeout);
            return { success: true, attempt };
        } catch (error) {
            lastError = error;
            console.error(`❌ 尝试 ${attempt} 失败：${error.message}`);
            
            if (attempt <= config.retry) {
                console.log(`⏳ ${config.retryDelay / 1000}秒后重试...`);
                await sleep(config.retryDelay);
            }
        }
    }
    
    return { 
        success: false, 
        error: lastError.message,
        attempts: config.retry + 1
    };
}
```

#### 4.2 错误分类

| 错误类型 | 处理方式 | 示例 |
|---------|---------|------|
| **网络错误** | 自动重试 3 次 | API 超时、连接失败 |
| **数据错误** | 记录警告，继续执行 | 某只股票数据缺失 |
| **脚本错误** | 终止工作流 | 语法错误、模块缺失 |
| **权限错误** | 发送警报 | 文件无法写入 |
| **资源错误** | 清理后重试 | 磁盘空间不足 |

#### 4.3 降级策略

```javascript
// 如果主 API 失败，切换到备用 API
async function fetchStockData(tsCode) {

# 创建工作流定时任务（替代原有的数据更新任务）
$action = New-ScheduledTaskAction -Execute "node" `
    -Argument "run-full-workflow.js" `
    -WorkingDirectory "C:\Users\Bob\.openclaw\workspace\stock-data"

$trigger = New-ScheduledTaskTrigger -Weekly `
    -DaysOfWeek Monday,Tuesday,Wednesday,Thursday,Friday `
    -At 4pm

$settings = New-ScheduledTaskSettingsSet `
    -AllowStartIfOnBatteries `
    -DontStopIfGoingOnBatteries `
    -StartWhenAvailable `
    -ExecutionTimeLimit (New-TimeSpan -Hours 1)

Register-ScheduledTask `
    -TaskName "StockAnalysisWorkflow" `
    -TaskPath "\Stock\" `
    -Action $action `
    -Trigger $trigger `
    -Settings $settings `
    -RunLevel Highest `
    -Force
```
